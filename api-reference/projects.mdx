# Projects

Manage your SpecStory workspaces and organize your AI conversations. Projects serve as containers for your AI chat sessions, allowing you to organize conversations by codebase, feature, or any logical grouping that makes sense for your workflow.

<Tabs>
  <Tab title="List Projects">
    ## GET /api/v1/projects

    List all projects in your SpecStory workspace.

    ### Request

    <CodeGroup>

    ```bash cURL
    curl -H "Authorization: Bearer YOUR_API_KEY" \
      https://cloud.specstory.com/api/v1/projects
    ```

    ```javascript JavaScript
    const response = await fetch('https://cloud.specstory.com/api/v1/projects', {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      }
    });

    const data = await response.json();
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }

    response = requests.get(
        'https://cloud.specstory.com/api/v1/projects',
        headers=headers
    )

    data = response.json()
    ```

    </CodeGroup>

    ### Response

    ```json
    {
      "success": true,
      "data": {
        "projects": [
          {
            "id": "proj_123",
            "ownerId": "user_456",
            "name": "My Web App",
            "icon": "Code2",
            "color": "blue",
            "createdAt": "2024-01-15T10:30:00Z",
            "updatedAt": "2024-01-15T10:30:00Z"
          },
          {
            "id": "proj_789",
            "ownerId": "user_456", 
            "name": "Mobile App",
            "icon": "Rocket",
            "color": "emerald",
            "createdAt": "2024-01-10T14:20:00Z",
            "updatedAt": "2024-01-12T09:15:00Z"
          }
        ],
        "total": 2
      }
    }
    ```

    ### Response Fields

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | string | Unique project identifier |
    | `ownerId` | string | ID of the project owner |
    | `name` | string | Human-readable project name |
    | `icon` | string | Project icon identifier (see available icons below) |
    | `color` | string | Project color theme |
    | `createdAt` | string | ISO 8601 timestamp of project creation |
    | `updatedAt` | string | ISO 8601 timestamp of last update |

    ### Available Icons

    Projects support the following icon options: `Code2`, `Terminal`, `Bot`, `Brain`, `Sparkles`, `Database`, `GitBranch`, `Bug`, `FolderKanban`, `Folder`, `Files`, `Library`, `BookOpen`, `Briefcase`, `Package`, `Layers`, `AppWindow`, `Rocket`, `Palette`, `Paintbrush`, `PenTool`, `Lightbulb`, `BarChart3`, `Settings`, `Globe`

    ### Available Colors

    Projects support these color themes: `rose`, `pink`, `orange`, `amber`, `emerald`, `cyan`, `blue`, `purple`, `slate`, `gray`

  <Tab title="Update Project">
    ## PATCH /api/v1/projects/{projectId}

    Update a project's name, icon, or color. You can update any combination of these fields in a single request.

    ### Request

    <CodeGroup>

    ```bash cURL
    curl -X PATCH \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"name": "Updated Project Name", "icon": "Rocket", "color": "emerald"}' \
      https://cloud.specstory.com/api/v1/projects/proj_123
    ```

    ```javascript JavaScript
    const response = await fetch('https://cloud.specstory.com/api/v1/projects/proj_123', {
      method: 'PATCH',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        name: 'Updated Project Name',
        icon: 'Rocket',
        color: 'emerald'
      })
    });

    const data = await response.json();
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }

    payload = {
        'name': 'Updated Project Name',
        'icon': 'Rocket',
        'color': 'emerald'
    }

    response = requests.patch(
        'https://cloud.specstory.com/api/v1/projects/proj_123',
        headers=headers,
        json=payload
    )

    data = response.json()
    ```

    </CodeGroup>

    ### Request Body

    | Field | Type | Required | Description |
    |-------|------|----------|-------------|
    | `name` | string | No | New project name |
    | `icon` | string | No | New project icon (see available icons in List Projects) |
    | `color` | string | No | New project color (see available colors in List Projects) |

    ### Response

    ```json
    {
      "success": true,
      "data": {
        "name": "Updated Project Name",
        "icon": "Rocket",
        "color": "emerald"
      }
    }
    ```

    ### Error Responses

    **403 Forbidden** - You don't have permission to update this project:
    ```json
    {
      "success": false,
      "error": "Insufficient permissions"
    }
    ```

    **404 Not Found** - Project doesn't exist:
    ```json
    {
      "success": false,
      "error": "Project not found"
    }
    ```

  <Tab title="Delete Project">
    ## DELETE /api/v1/projects/{projectId}

    Permanently delete a project and all its associated sessions.

    <Warning>
      **This action cannot be undone.** All sessions within the project will be permanently deleted along with the project itself.
    </Warning>

    ### Request

    <CodeGroup>

    ```bash cURL
    curl -X DELETE \
      -H "Authorization: Bearer YOUR_API_KEY" \
      https://cloud.specstory.com/api/v1/projects/proj_123
    ```

    ```javascript JavaScript
    const response = await fetch('https://cloud.specstory.com/api/v1/projects/proj_123', {
      method: 'DELETE',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    });

    const data = await response.json();
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY'
    }

    response = requests.delete(
        'https://cloud.specstory.com/api/v1/projects/proj_123',
        headers=headers
    )

    data = response.json()
    ```

    </CodeGroup>

    ### Response

    ```json
    {
      "success": true,
      "data": {
        "deletedProject": {
          "id": "proj_123",
          "name": "My Web App"
        },
        "deletedAt": "2024-01-15T11:00:00Z"
      }
    }
    ```

    ### Response Fields

    | Field | Type | Description |
    |-------|------|-------------|
    | `deletedProject.id` | string | ID of the deleted project |
    | `deletedProject.name` | string | Name of the deleted project |
    | `deletedAt` | string | ISO 8601 timestamp when deletion occurred |

    ### Error Responses

    **403 Forbidden** - You don't have permission to delete this project:
    ```json
    {
      "success": false,
      "error": "Insufficient permissions"
    }
    ```

    **404 Not Found** - Project doesn't exist:
    ```json
    {
      "success": false,
      "error": "Project not found"
    }
    ```

    ### Best Practices

    Before deleting a project, consider:
    - **Export important sessions** using the Sessions API if you need to preserve any conversation data
    - **Verify project contents** by listing sessions first to understand what will be deleted
    - **Use with caution** in production environments - consider implementing additional confirmation steps in your application

## Project Organization Tips

Projects in SpecStory work best when organized around logical boundaries:

- **By Codebase**: Create separate projects for different repositories or applications
- **By Feature**: Group conversations related to specific features or components
- **By Team**: Organize projects by team ownership or responsibility areas
- **By Environment**: Separate development, staging, and production-related conversations

The icon and color system helps you quickly identify projects at a glance, making it easier to navigate between different workspaces as your SpecStory usage grows.
